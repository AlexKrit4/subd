# Лабораторная работа 5

## Тема
Реализация процедур и триггеров для расширенной схемы.

## Принятый вариант
Вариант 3 -> задание 3: замена типа самолета на рейс с проверкой вместимости.

## Что реализовано
В файле [procedures.sql](procedures.sql):
1. `assign_employee_seat(...)` — посадка сотрудника без билета.
2. `register_inflight_seat_change(...)` — регистрация смены места пассажира в полете.
3. `replace_aircraft_with_check(...)` — безопасная замена типа ВС с проверкой проданных билетов.
4. Триггер `trg_loyalty_recalc` для автоматического пересчета баланса миль.

## Ключевые проверки
1. Проверка существования рейса и места в конфигурации самолета.
2. Контроль конфликта занятости мест (пассажир/сотрудник).
3. Запрет замены самолета при недостатке мест.

## Итог
Подготовлен набор процедур, который используется в прикладной части (Lab 7).
